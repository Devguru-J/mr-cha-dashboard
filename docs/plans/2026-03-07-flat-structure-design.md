# Flat Structure Refactoring Design

Date: 2026-03-07
Status: Approved

## Problem

The current monorepo structure (`apps/web/` + `functions/api/` as separate workspace packages) adds
unnecessary complexity for a single-app, single-API project. Issues:

- Three separate `node_modules` directories (root, apps/web, functions/api)
- Both `package-lock.json` (npm) and `bun.lock` (bun) coexist ‚Äî package manager ambiguity
- Cloudflare Pages Functions bundler works best with a single root `node_modules`
- No multi-app benefit (only one app under `apps/`)

## Decision

Migrate to a flat, single-package structure identical to the previous project layout. This aligns
with Cloudflare Pages Functions conventions and eliminates workspace complexity.

Package manager: **Bun** (standardize, remove all `package-lock.json` files)

## Files Deleted

- `apps/` directory (entire)
- `functions/api/package.json`
- `functions/api/tsconfig.json`
- `functions/api/package-lock.json`
- Root `package-lock.json`

## Config File Updates

| File | Change |
|------|--------|
| `wrangler.toml` | `pages_build_output_dir = "apps/web/dist"` ‚Üí `"dist"` |
| `.gitignore` | `apps/web/dist` ‚Üí `/dist`, remove `apps/web/node_modules` entries |
| `drizzle.config.ts` | Update schema/out paths: `functions/api/drizzle/` ‚Üí `drizzle/` |

## Implementation Order

1. Commit & push current state (snapshot before refactor)
2. Move frontend files: `apps/web/src` ‚Üí `src/`, `apps/web/public` ‚Üí `public/`, etc.
3. Move API support files: `functions/api/drizzle/` ‚Üí `drizzle/`, `drizzle.config.ts` ‚Üí root
4. Create merged root `package.json`
5. Update `wrangler.toml`, `.gitignore`, `drizzle.config.ts`
6. Delete `apps/`, remove workspace `package.json` files, remove `package-lock.json` files
7. Run `bun install` to regenerate `bun.lock`
8. Verify: `bun run build` succeeds
9. Verify: `bun run dev` starts both services
